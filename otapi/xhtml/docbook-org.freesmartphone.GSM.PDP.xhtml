ActivateContext ()
ActivateContext (in  's' apn,
                 in  's' username,
                 in  's' password,
                 out 'i' index)

        Request a PDP context activation to an IP-based network service provider and launch a ppp session.
      
apn:
The access point name to connect to.
username:
The user name to authenticate with.
password:
The password to authenticate with.
index:
An index to identify context status changes.
Implementation notes:

        This can map to the following GSM 07.07 commands:
        * +CGDCONT (see 3GPP TS 07.07 chapter 10.1.1),
        * +CGATT (see 3GPP TS 07.07 chapter 10.1.4),
        * +CGACT (see 3GPP TS 07.07 chapter 10.1.5),
        * +CGDATA (see 3GPP TS 07.07 chapter 10.1.6).
        It might also map to the v.250 compatibility commands D*...
      
Implementation notes:
This command might change your network default route.

Signal Details
The ContextStatus signal
ContextStatus ('i'     id,
               's'     status,
               'a{sv}' properties)

        Sent whenever there is any status or property change on a pdp context in the system.
      
id:
The index of the context that changed its status or properties.
status:

          The new status of the call. Expected values are:
          * "incoming" = The context is incoming (but not yet accepted),
          * "outgoing" = The context is outgoing (but not yet established),
          * "active" = The context is active (you have a new default route),
          * "held" = The context is being held (interrupted by a voice call),
          * "released" = The context is being released.
          Further information can be part of the properties.
        
properties:

          An array of property values. Note that properties are optional.
          Expected tuples are:
          * ( "reason", string ) = The reason for the status change,
          * ( "apn", string ) = The number of the access point,
